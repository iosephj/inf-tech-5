Es decir...

- *“No importa que funcione, importa que entiendas por qué”*

## 🧪 Diagnóstico inicial — Informática

Si no conocés el lenguaje, intentá deducir qué hace el código. No se evalúa memoria sino razonamiento.


**1. ¿Qué imprime este código?**

```python
x = 5
if x > 3:
    print("A")
else:
    print("B")
```

A) A
B) B
C) Error
D) Nada


**2. ¿Qué hace este código?**

```python
for i in range(3):
    print(i)
```

A) Imprime 1, 2, 3
B) Imprime 0, 1, 2
C) Imprime 0, 1, 2, 3
D) Da error


**3. ¿Cuál es el resultado?**

```python
a = 10
b = 3
print(a // b)
```


A) 3.33
B) 3
C) 4
D) Error


**4. ¿Qué representa mejor una variable?**

A) Un número fijo
B) Un lugar donde se guarda un valor
C) Una operación matemática
D) Un tipo de dato


**5. ¿Qué es un algoritmo?**

A) Un lenguaje de programación
B) Una computadora
C) Una serie de pasos para resolver un problema
D) Un programa ya hecho



**6. Si una fórmula en una planilla da mal, lo más probable es que:**

A) La computadora esté rota
B) El programa esté mal diseñado
C) El usuario cometió un error en los datos o fórmula
D) Es imposible saber



**7. ¿Cuál de estas es una condición válida?**

A) `x = 5`
B) `x == 5`
C) `5 x`
D) `=x5`



**8. ¿Qué hace un bucle (loop)?**

A) Guarda datos
B) Repite acciones
C) Compara valores
D) Muestra resultados

“Todos los tornillos son pesados”?

C) No se puede saber
